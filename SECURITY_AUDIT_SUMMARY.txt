KEY FINDINGS:
=============

CRITICAL (1):
- Issue #1: Prompt Injection in Wiki Articles
  File: src/generators/wiki.ts lines 268-290, src/formatter.ts

  Route paths, middleware names, and env vars are interpolated into markdown
  WITHOUT escaping backticks/brackets/asterisks. Attackers can inject
  markdown syntax that breaks code fences or adds instructions.

  Fix Time: 1-2 hours
  Status: REQUIRES IMMEDIATE ACTION

HIGH SEVERITY (2):
- Issue #2: MCP File Path Traversal
  File: src/mcp-server.ts lines 184-209
  toolGetBlastRadius() accepts file paths without validation.
  Could leak filesystem information across project boundaries.
  Fix Time: 30 minutes

- Issue #3: Environment Variable Disclosure
  File: src/generators/wiki.ts line 235
  Sensitive env var names (containing KEY, SECRET, TOKEN, PASSWORD)
  are exposed in wiki and MCP tool responses.
  Fix Time: 1 hour

MEDIUM SEVERITY (4):
- Issue #4: Schema Field Name Injection (Database models not escaped)
- Issue #5: Component Props Injection (React/Vue props not escaped)
- Issue #6: Middleware Names in Routes (Low impact but related to #1)
- Issue #7: Redundant Detector Execution (10-15% token waste)

LOW SEVERITY (3):
- Issue #8: Binary File Scanning (3-5% token waste)
- Issue #9: Monorepo Schema Duplication (20-30% monorepo token waste)
- Issue #10: Statistics Output Unbounded (DoS edge case)

calling Claude API directly) is a security advantage. This prevents
direct injection at the API boundary.
